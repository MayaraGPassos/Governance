The Transformation Journey

Why Governance Transformation is Necessary

In a rapidly evolving global R&D landscape, efficiency and agility are no longer options—they are survival requirements. Our transformation initiative is designed to address systemic operational challenges, replace fragmented practices with unified standards, and build a culture of high performance.

Current Challenges

  • •

    Lack of Visibility: Fragmented project schedules make it impossible to track progress in real-time across centers.

  • •

    Unpredictability: Delays are detected too late, preventing timely escalation and proactive risk mitigation.

  • •

    Capacity Constraints: No clear visibility into team workload, leading to overallocation or hidden bottlenecks.

Future State Vision

  • •

    Unified Standards: One single source of truth for all projects, tasks, and timelines globally.

  • •

    High Predictability: Standardized escalation triggers (Tiers) and automated delay evaluation.

  • •

    Optimized Capacity: Real-time workload tracking (Work Log) supporting strategic decision-making.

M365 Licensing & Access Setup

Before using any governance tools, all team members must be assigned appropriate licenses and permissions. Follow these critical setup steps.

1

License Check: Verify that the user has a valid Microsoft 365 license (F3, E3, or E5) assigned in the Admin Center.

2

Access Request Form: Open the "M365 Resource Request" form, enter user details, and select the required applications (Planner, Teams, SharePoint).

3

Manager Approval: Submit the form. The system routes the request to the direct manager for formal approval before licensing is active.

Microsoft Planner Task Management

MS Planner acts as our visual task management engine. Standardizing buckets and labels is key to ensuring cross-site data aggregation works seamlessly.

1

Bucket Standardization: Organize boards into standardized buckets: Backlog, Planned, In Progress, Review, and Completed.

2

Label Conventions: Use color-coded labels to indicate task types, critical dependencies, or risk levels (e.g., Red for Blocked, Yellow for At Risk).

3

Task Assignment: Every task must have a single assigned owner, a clear start date, and a realistic due date aligned with the master schedule.

Work Log App & Capacity Tracking

The Work Log App is used by analysts and team members to log actual hours worked against planned tasks, giving us real-time capacity insights.

1

Daily Logging: Analysts must log hours daily. Select the project, the corresponding Planner task, and input the hours spent.

2

Activity Classification: Classify hours correctly (e.g., Development, Review, Admin, Training) to ensure accurate capacity modeling.

3

Validation & Submission: Submit logs at the end of each week. Leaders review and validate submissions before they feed into Power BI.

Work Log Screen Reference
Work Log PowerApp Interface Screenshot Ref: Time Submission Portal

Power BI Dashboards & Analytics

Our Power BI dashboards pull data from Planner and Work Log, presenting unified, real-time performance and capacity metrics to leadership.

1

Data Sync Frequency: Dashboards are set to auto-refresh 4 times daily, ensuring leadership has access to near-real-time metrics.

2

Key Metrics Tracked: Monitor Schedule Variance, Analyst Allocation Rate, Task Lead Time, and Postponement Offenders.

3

Access Control: Dashboards are shared via workspace groups. Leadership and PMO have edit rights, while team members have read-only access.

PMO Step EIB BC Approved

This is the formal trigger for any new project in the R&D center. Once the business case is approved, the PMO initiates the governance onboarding.

PMO Step Kick-Off Meeting

The PMO aligns with the project leader and key stakeholders to establish the project baseline, defining critical milestones and deliverables.

PMO Step PMO Evaluation (Tier 2)

During monthly Tier 2 meetings, the PMO evaluates the impact of any timeline changes or capacity constraints, triggering escalations if necessary.

Gov. Owner Step New Project Start Flow

The Governance Owner initiates the system flow by creating the standard project form, setting up the SharePoint repository and MS Planner board.

Gov. Owner Step Standard Timeline Update

The Governance Owner ensures that the project schedule in MS Planner is kept up to date, feeding directly into the global Power BI analytics.

Gov. Owner Step Monthly Status Report

Generates the consolidated monthly status report for leadership, detailing capacity bottlenecks, postponements, and schedule adherence.

Leadership Step Scope Definition

The Site Leader and Project Manager define the specific technical scope and deliverables required for the project.

Leadership Step Timeline Validation

Validates the planned project timeline in MS Planner, ensuring resource allocation is realistic before committing to the schedule.

Leadership Step Project Tasks Follow-up

Conducts weekly follow-ups with the team to review task progress, resolve operational blockers, and approve postponement requests.

Team Step Project Tasks Received

Team members receive their assigned tasks in MS Planner, reviewing the requirements, start dates, and due dates.

Team Step Project Tasks Updates

Team members update their task status in MS Planner and log their daily actual hours in the Work Log App.
